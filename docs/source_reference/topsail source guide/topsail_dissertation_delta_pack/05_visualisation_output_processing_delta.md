## Output-processing rules

1. **CSV trajectories are the canonical diagnostic substrate.** Source: Diss p. 216 (PDF p. 251), §4.2.6 and Diss p. 245 (PDF p. 280), §6.2.5. Use CSV exports for downstream Python dashboards instead of scraping MATLAB plots.
2. **Plot generation must be optional for first four-bed validation.** Source: Diss p. 243 (PDF p. 278), §6.2.4. [INFERENCE] Since >2 adsorbers are not currently supported and plotting may contain two-bed assumptions, simulation correctness should not depend on plotting.
3. **Every report should label dimensional versus dimensionless quantities.** Source: Diss p. 165-166 (PDF p. 200-201), §3.7.1 and Diss p. 189-191 (PDF p. 224-226), §3.7.6. This prevents ledgers from adding scaled and physical moles, a surprisingly efficient path to nonsense.
4. **Event plots must show both selected and suppressed objectives for multi-bed cases.** Source: Diss p. 235-236 (PDF p. 270-271), §5.5.3. [INFERENCE] This is the only practical way to audit event-priority decisions in four-bed cycles.
