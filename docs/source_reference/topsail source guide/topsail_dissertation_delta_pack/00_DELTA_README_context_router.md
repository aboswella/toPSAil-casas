# 00 - Delta README and context router

This mini-pack extracts dissertation-only deltas from Taehun Kim's PhD dissertation, *Computational Methods for Intensifying the Design and Operation of Pressure Swing Adsorption Processes*. It is designed to supplement, not replace, the existing split toPSAil paper extraction pack. The point is to stop future agents from rediscovering the same six facts while pretending the CPU fan is scholarship.

Use these files only when the dissertation adds implementation detail, a stronger limitation, a new numerical condition, a workflow anchor, or a future-development priority.

## Carry-forward delta principles

- The dissertation refines the earlier four-bed assumption: current toPSAil is described as not supporting more than two adsorbers, not merely as having a two-bed UI limit. See `07` and `09`.
- Treat fixed-composition literature streams as source/sink semantics, not as initial tank compositions. The dissertation adds MFC and product-tank outlet control laws, but not a magic reservoir that stays pure because a spreadsheet once believed in itself.
- Before parameter tuning, add ledgers for tanks, external products, cumulative flows, events, and CSS. Tuning before accounting is just curve-fitting with stationery.
- Do not implement gL, MS-LDF, S-shaped isotherms, or hysteresis by aliasing an existing explicit model. The dissertation labels these as future implicit-model work.
