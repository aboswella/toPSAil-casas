FOUR-BED toPSAil IMPLEMENTATION: LEGACY WORKFLOW CONTEXT FILE MAP

Purpose
This file used to route future ChatGPT/Codex-planning instances through the
WP1-WP5 workflow CSVs. That workflow is now legacy context.

For active final implementation work, start with:

1. docs/four_bed/README.md
2. docs/four_bed/FINAL_IMPLEMENTATION_CONTEXT.md
3. the relevant active batch guide under docs/four_bed/

The active final target is a Yang-inspired H2/CO2 homogeneous activated-carbon
surrogate. The active task sequence is FI-1 through FI-8 grouped into six final
implementation batches. Do not use the old WP1-WP5 CSVs as the current task
sequence unless the user explicitly asks for legacy review.

Legacy CSV files

The CSV files in this folder remain useful for historical rationale, old issue
IDs, old test IDs, source anchors, and architecture-risk cross-checks:

0. four_bed_executive_summary.csv
   Historical high-level corrected architecture summary.

1. four_bed_work_packages.csv
   Legacy WP1-WP5 scope and handoff tests.

2. four_bed_architecture_map.csv
   Historical architecture constraints. Its no-tank, no-header, no-global-RHS,
   and no-core-physics-rewrite guardrails still apply.

3. four_bed_yang_manifest.csv
   Legacy Yang Table 2 schedule skeleton. Use for source-label history only;
   executable final timing is normalized by the final implementation context.

4. four_bed_issue_register.csv
   Legacy issue register. Use for risk cross-checking when a failure mode still
   applies.

5. four_bed_test_matrix.csv
   Legacy test matrix. Use old test IDs for traceability when they still match
   active final tests.

6. four_bed_stage_gates.csv
   Legacy development sequence. It no longer defines the active final batch
   order.

7. four_bed_evidence_notes.csv
   Historical source anchors and rationale.

8. Work package guidance docs/WP1_yang_schedule_manifest_guidance.md
   Retired WP1 implementation guidance. Do not use it as active scope.

Conflict policy

If these legacy files conflict with docs/four_bed/FINAL_IMPLEMENTATION_CONTEXT.md
or an active batch guide, use the final implementation context and report the
contradiction in the task report. Do not silently revive old work-package scope.
