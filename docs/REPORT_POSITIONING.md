# Report Positioning

## Core Report Claim

Use this position unless later evidence changes the project direction:

> The simulator uses toPSAil's native bed-step machinery as the numerical engine
> wherever suitable. The Yang four-bed work is a thin wrapper-and-adapter layer
> that schedules persistent named bed states, applies explicit direct-transfer
> pair maps, runs custom PP->PU and AD&PP->BF adapters where native step grammar
> is insufficient, and reconstructs external/internal ledgers without adding
> dynamic internal tanks, shared header inventory, or a global four-bed RHS.

## Active Model Language

The active final implementation is a Yang-inspired H2/CO2 homogeneous
activated-carbon surrogate.

Say plainly that it:

- uses binary H2/CO2 feed renormalization;
- uses activated carbon as a homogeneous adsorbent;
- uses normalized executable schedule durations
  `[1,6,1,4,1,1,4,1,1,5]/25`;
- excludes zeolite 5A, CO, CH4, pseudo-components, and layered-bed behaviour in
  the first final implementation;
- reconstructs final H2 purity and recovery from wrapper external-stream rows.

## Active Implementation Language

The old WP1-WP5 language is legacy. Current reports should refer to:

- FI-1 schedule finalisation;
- FI-2 H2/CO2 AC parameter pack;
- FI-3 physical-state persistence cleanup;
- FI-4 PP->PU adapter;
- FI-5 AD&PP->BF adapter;
- FI-6 four-bed cycle driver;
- FI-7 ledger extraction and audit output;
- FI-8 commissioning tests.

Legacy WP labels may be mentioned only for historical traceability.

## What Not To Claim

Do not claim physical Yang validation merely because:

- the source schedule was transcribed;
- static manifest tests pass;
- a wrapper executes one cycle;
- native metrics are available;
- a homogeneous H2/CO2 surrogate runs;
- pressure endpoints can be tuned with valve coefficients.

## Required Caveats

Reports must address:

- source traceability;
- H2/CO2 renormalization;
- homogeneous activated-carbon surrogate basis;
- architecture constraints;
- direct-transfer pair-map status;
- persistent physical-state writeback status;
- external/internal ledger basis;
- adapter pressure and conservation diagnostics;
- thermal assumptions;
- numerical health;
- validation agreement only after earlier commissioning gates pass.
